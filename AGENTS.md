# Spark Forge AI — Agent Guide

## Quick start

```bash
npm install
npm run dev          # vite dev server on port 8080
npm run build        # vite build && copy output to output/
npm run preview      # vite preview
npm run lint         # ESLint + Prettier
npx prettier --check src/
npx tsc --noEmit     # typecheck
```

- No test framework exists (no test scripts, no test files).
- `lint` then `tsc` then `build` is the safe verification order.

## Architecture

- **Framework**: [TanStack React Start](https://tanstack.com/router/latest) — SSR, file-based routing under `src/routes/`, server functions in `src/lib/*.functions.ts`.
- **Entrypoints**: `src/server.ts` (Cloudflare/Wrangler), `src/start.ts` (TanStack Start instance), `src/router.tsx` (router creation).
- **`src/routeTree.gen.ts`** is auto-generated by TanStack Router — never edit manually.
- **Packages**: single package (no monorepo).
- **Deployment**: Vercel (`vercel.json`, Nitro output) or Cloudflare Workers (`wrangler.jsonc`).

## Key integrations

- **Supabase**: auth (Row Level Security), database, caching. Client at `src/integrations/supabase/client.ts`, server admin client at `client.server.ts`.
- **AI gateway**: `src/lib/ai-gateway.ts` — multi-provider with failover, 5-min cooldown, 30–60s timeout. Supports NVIDIA, DeepSeek, Cohere, OpenAI, Gemini, Anthropic, X.ai, OpenRouter, Groq, ProjectSpark, mock.
- **3D scenes**: React Three Fiber via `@react-three/fiber`, `@react-three/drei`, `@react-three/rapier`. Scene state in a shared Zustand store (`useSceneStore`).
- **shadcn/ui** (new-york style) at `src/components/ui/`. Add components via `npx shadcn@latest add <name>`.

## Code conventions

- **Path alias**: `@/*` → `./src/*` (configured in tsconfig.json and vite.config.ts).
- **Styling**: Tailwind CSS v4, OKLCH color space, dark-first, class-variance-authority + `cn()` from `src/lib/utils.ts`.
- **Formatter**: Prettier — `printWidth: 100`, `semi: true`, `singleQuote: false` (double quotes), `trailingComma: "all"`. Ignores `routeTree.gen.ts` and lockfiles.
- **Linter**: ESLint — no unused vars rule is **off**; blocks `server-only` imports; `react-refresh/only-export-components` at warn level.
- **No React compiler or RSC** (`components.json`: `"rsc": false`).
- **Env**: `.env.local` for secrets (gitignored). See `.env.local` for expected vars (ANTHROPIC_API_KEY, SUPABASE_*, etc.).
- **VSCode**: `css.validate: false` (built-in CSS validation disabled for Tailwind).

## Project quirks

- **Vite config** (`vite.config.ts`): plugin order matters — tailwindcss → tsconfigPaths → tanstackStart → nitro → react → visualizer. Manual chunks for three.js, gsap, framer-motion, recharts, ai-sdk. Server on IPv6 `::` port 8080.
- **Post-build**: `node copy-output.js` copies `.vercel/output` to `output/`. Requires Nitro output existence.
- **Bun supply-chain guard**: `bunfig.toml` enforces 24h minimum release age (exempts `@lovable.dev/vite-tanstack-config`).
- **Scratch scripts** in `scratch/` test AI provider keys — not part of the app.
- **`.prettierignore`**: also ignores `pnpm-lock.yaml`, `package-lock.json`, `bun.lock`.
